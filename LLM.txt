## 2026-03-31 — Phase 2: Remove Redundant CSS Fallback Values

### What was done
- Removed 211+ redundant `var(--moka-TOKEN, FALLBACK)` inline fallbacks from 39 `.razor.css` files
- Three pattern types cleaned: `#hex` fallbacks, `rgba()` fallbacks, and `var()` token-to-token fallbacks
- Preserved component-local custom properties (e.g., `--moka-swatch-columns`, `--moka-popover-offset-x`)
- All tokens are defined in `moka.css` `:root`, so removing inline fallbacks is safe

### Why this matters
- Single source of truth: changing a token value in `moka.css` or via `MokaTheme` now takes effect everywhere
- No more drift risk where inline fallbacks silently preserve stale values
- Cleaner CSS that's easier to read and maintain

### Build & test results
- `dotnet build Moka.Red.slnx` — 0 warnings, 0 errors
- 124 tests pass (67 Core + 57 Diagnostics)
- Browser verified: all components render correctly with colors, spacing, and typography intact
